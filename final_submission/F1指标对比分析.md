## 📊 Macro F1 vs Weighted F1 详细对比

### Weighted F1 (加权F1)

**计算方式**：
```
Weighted F1 = (F1_casual × n_casual + F1_member × n_member) / n_total
            = (0.4740 × 13,014 + 0.8251 × 33,812) / 46,826
            = 0.7275
```

**特点**：
- ✅ 按样本数量加权
- ✅ 反映整体分类性能
- ✅ 适合样本不平衡的场景
- ❌ 会被多数类主导

**解读**：
- 这个版本的Weighted F1=0.7275，因为Member类（占72%）表现很好（F1=0.8251）
- Casual类表现较差（F1=0.4740），但因为样本少（占28%），对总分影响小

### Macro F1 (宏平均F1)

**计算方式**：
```
Macro F1 = (F1_casual + F1_member) / 2
         = (0.4740 + 0.8251) / 2
         = 0.6496
```

**特点**：
- ✅ 每个类别权重相同
- ✅ 反映各类别平均性能
- ✅ 不受样本分布影响
- ✅ 更公平地评估模型

**解读**：
- Macro F1=0.6496，反映了模型在两类上的平均表现
- 揭示了模型对Casual类识别能力较弱的问题

## 🔍 为什么两个F1差距这么大？

### 原因分析

**1. 样本不平衡**
```
Member: 33,812样本 (72.2%)
Casual: 13,014样本 (27.8%)

比例：Member : Casual = 2.6 : 1
```

**2. 模型偏向多数类**
```
Member类：
  Recall=85.76% (很好地识别了会员)
  Precision=79.50% (预测为会员的准确率高)
  F1=82.51%

Casual类：
  Recall=42.56% (只识别了42%的散户)
  Precision=53.49% (预测为散户的准确率低)
  F1=47.40%
```

**3. Weighted F1被Member类拉高**
```
Weighted F1 = 0.4740×0.278 + 0.8251×0.722
            = 0.132 + 0.596
            = 0.7275

Member类贡献了82%的加权分数！
```
