### 2.4 数据质量分析

#### 2.4.1 数据清洗过程

为保证数据质量，本实验对抽样数据进行了严格的清洗，具体步骤如下：

| 清洗步骤 | 样本数变化 | 删除条数 | 删除率 |
|----------|------------|----------|--------|
| 初始抽样 | 120,000 → 60,000 | 60,000 | 50.0% |
| 有效目标类别 | 60,000 → 60,000 | 0 | 0.0% |
| 有效时间戳 | 60,000 → 60,000 | 0 | 0.0% |
| 有效坐标范围 | 60,000 → 59,939 | 61 | 0.1% |
| DC区域范围内 | 59,939 → 59,939 | 0 | 0.0% |
| 时长1-180分钟 | 59,939 → 58,416 | 1,523 | 2.5% |
| 合理距离和速度 | 58,416 → 58,405 | 11 | 0.02% |
| 去重复ride_id | 58,405 → 58,405 | 0 | 0.0% |

**清洗规则说明**：
1. **时间有效性**：started_at < ended_at，时长在1-180分钟
2. **空间有效性**：坐标在DC区域范围内（38.79°N-39.0°N, 77.12°W-76.91°W）
3. **速度合理性**：骑行速度在2-30 km/h之间
4. **数据唯一性**：删除重复的ride_id

#### 2.4.2 数据分布特征

**时间分布**：
- 12个月样本分布均匀，每月约4,867条记录
- 工作日占比：73.0%，周末占比：27.0%
- 高峰时段：17点（晚高峰）骑行量最大

**空间分布**：
- 起点集中在市中心和交通枢纽
- 平均骑行距离：2.8 km（会员），3.1 km（散户）
- 平均骑行时长：11.5分钟（会员），17.5分钟（散户）

**用户行为差异**：

| 特征 | 会员（Member） | 散户（Casual） | 差异 |
|------|----------------|----------------|------|
| 平均时长 | 11.54分钟 | 17.51分钟 | +52% |
| 中位时长 | 8.59分钟 | 11.04分钟 | +29% |
| 周末占比 | 25.11% | 34.99% | +39% |
| 通勤高峰占比 | 33.81% | 25.19% | -26% |
| 环线骑行占比 | 5.29% | 6.36% | +20% |

**关键发现**：
- 散户骑行时间明显更长，符合休闲、旅游使用场景
- 会员工作日通勤高峰占比更高，符合通勤使用场景
- 散户周末骑行比例更高，说明休闲属性更强
