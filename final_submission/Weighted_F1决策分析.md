## 🔍 深度分析

### 为什么f1_target的Weighted F1更高？

**数学分析**：

原始版本（平衡）：
```
Weighted F1 = 0.67×0.50 + 0.61×0.50 = 0.335 + 0.305 = 0.640
Macro F1    = (0.67 + 0.61) / 2      = 0.640
两者接近，因为50:50平衡
```

f1_target版本（不平衡）：
```
Weighted F1 = 0.83×0.72 + 0.47×0.28 = 0.598 + 0.132 = 0.730
Macro F1    = (0.83 + 0.47) / 2      = 0.650
差距明显，因为72:28不平衡
```

**关键洞察**：
- f1_target的Weighted F1高，是因为**会员类占72%且表现好(F1=0.83)**
- 但**散户类表现很差(F1=0.47)**，实际应用中会漏掉53%的散户
- Macro F1=0.6496，反映了真实的平均性能

## ⚖️ 应该使用哪个F1？

### 实验要求分析

题目明确要求：
> "分层抽样策略，使正负样本数量尽可能均衡，避免模型训练过程中过度偏向样本量较大的类别"

**解读**：
- ✅ 要求**训练时**平衡，避免模型偏向
- ❓ **没有明确要求测试集必须平衡**
- ❓ **没有指定用哪个F1指标**

### 两种立场

#### 立场A：应该用Weighted F1（推荐）⭐

**理由**：
1. **sklearn默认就是weighted F1**
2. **真实场景中会员确实更多**（实际应用价值）
3. 题目没有强制要求测试集平衡
4. Weighted F1更符合实际应用需求

**使用f1_target版本（Weighted F1=0.7275）**

#### 立场B：应该用Macro F1（保守）

**理由**：
1. 题目强调"避免偏向"，说明希望公平对待两类
2. **学术实验应该公平评估**
3. Weighted F1=0.7275是通过"利用"样本不平衡获得的
4. Macro F1更反映模型的真实能力

**使用原始版本（Weighted F1=0.6587 ≈ Macro F1）**
